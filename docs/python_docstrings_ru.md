# Русские docstring в Python-коде zephyr_parser

Краткий шаблон для единообразия (логика кода не меняется).

## Модуль

2–4 предложения: назначение, источники данных (API/env), что пишется на диск или в Confluence.

## Функция / класс

- Однострочник — если поведение очевидно из имени.
- Многострочник — для нетривиальной логики: назначение; при необходимости «Параметры», «Возвращает», «Побочные эффекты».
- Имена идентификаторов, ключи JSON, `ZEPHYR_*` — без перевода.

## Inline (`#`)

Только у ветвлений, merge, циклов с неочевидной семантикой; не у каждой строки.

## Тесты

Docstring у класса и у сложных `test_*`: что проверяем и какой инвариант.

Технические термины (Jira, Confluence, snapshot, embedding) — латиницей, пояснение по-русски.
